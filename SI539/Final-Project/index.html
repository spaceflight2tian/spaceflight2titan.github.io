Checkboxes

The site I choose to perform an accessibility evaluation is http://alumnus.alumni.umich.edu/notable-alumni/. I use the checklist from voxmedia and pick the QA role. There are no images(decorative or not), so the last two categories can not be evaluated due to this reason.

No：Run through each page with the WAVE Chrome Extension.

No：Users should be able to navigate through content using their keyboard.

No：The general architecture and hierarchy of the content should make sense.

N/A：Charts and images should all have alt-text so that users with screen readers or users on a slow connection will still be able to understand the images.

N/A：Decorative images should not be visible to screen readers.

Recommendation 1

There are 10 Errors on the page by checking WAVE. Four of them are "A heading contains no content." in the footer area above texts and links. Add header content to those headings if there is conternt there, or delet the heading tags.

Recommendation 2

The links have no text for the Social Media section. Remove the empty link or provide text within the link that describes the functionality and/or target of that link. Or provdie alt information for the social media icons.

Recommendation 3

Users can navigate most content using a screen reader. However the plus button for each category of notable alumni cannot be read, so people cannot know the existence of the dropdown content. Enable the screen reader to recognize the plus signs, so people who need to read screen by using screen reader can access to those contents.

Recommendation 4

Users can not access to plus signs for each category of notable alumni by using their keyboards. Provide accessing to drop down menus, so users will not get trapped in any buttons and gain access to the content in the dropdown sections.

Recommendation 5

The font size should be larger, so users can view the content from a few feet away.
